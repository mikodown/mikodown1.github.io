冯诺伊曼计算机的特点
1945年，数学家冯诺依曼在研究EDVAC机时提出了存储结构的概念，以此概念为基础的计算机统称为冯诺依曼机，它的特点可以归结为：
1。计算机由运算器、存储器、控制器、输入设备和输出设备五大部件组成、
2.指令和数据以同等的地位存放于存储器中，并可按地址寻访
3.指令和数据均用二进制表示
4.指令由操作码和地址码组成，操作码用来表示操作的性质，地址码用来表示操作数在存储器中的位置
5.指令在存储器中按顺序存放，通常指令是顺序执行的，在特定条件下，可根据运算结果或根据设定的条件改变执行顺序。
6.机器以运算器为中心，输入输出设备与存储器之间的数据传输通过运算器完成.
典型的冯诺伊曼计算机结构图请另行查询课本
五大部件功能介绍
运算器用来玩长城算术运算和逻辑运算，并将运算中的中间结果暂时存放在运算器内。
存储器用来存放数据和程序
控制器用来控制和指挥程序和数据的输入，运行以及运算结果。
输入设备用来将人们熟悉的信息形式转换为机器能识别的信息形式，常见的有键盘，鼠标等。
输出设备可以用来将运算结果转换为人们熟悉的信息形式，如打印机、显示器输出等

        运算器和控制器在逻辑关系和电路的联系上十分紧密。尤其在大规模集成电路制作工艺出现后，这两大部件往往集成在一片芯片上因此，通常将他们合起来统称为中央处理器（cpu），把输入和输出设备简称为i/o设备

        这样现代的计算机可认为由三大部分组成，cpu i/o设备 及主存储设备

        cpu与主存储器合起来又可称为主机，输入设备和输出设备由可称为外被设备

    主存储器
        主存储器简称主存或内存包括存储体M，各种逻辑部件及控制电路等，存储体由许多存储单元组成，每个存储单元又包含若干个存储元件或称存储基元或存储元，每个存储元件能存储一位二进制代码 0或1 可见一个存储单元可存储一串二进制代码，称这串二进制代码为一个存储字，这串二进制代码的长度称为存储字长。
        存储字长可以是8位16位或32位等。可以赋予每个存储单元一个地址号。
        主存的工作方式就是按存储单元的地址号来实现对存储字的存取。这种方式称为按地址存取方式，即按照地址访问存储器。
    为了能实现按照地址访问的方式，主存中还必须配置两个寄存器MAR和MDR。MAR（memory address register）时存储器地址寄存器，用来存放欲访问第存储单元的地址号，其位数对应存储单元的个数，（如果MAR为10位，则有2^10=1024个存储单元，记为1K）MDR（memory data register）是存储数据寄存器，用来存放从存储体某单元取出的代码或者准备往某存储单元存入的代码，其位数与存储字长相同。


    64K * 32位的存储器，问MDR和MAR的位数各是多少？

        MDR为32位，MAR为16位
        MAR的位数代表字数，可以理解为存储单元数，如16位，代表2的16次方个存储单元，即64K个
        MDR的位数代表字长，如32位可以理解为4B（8位为1B)
        存储器容量=字数*字长 则上述容量为64K*4B=256KB

    运算器 
            运算器最少包含3个寄存器，（现代计算机中往往存在寄存器组）和一个算术逻辑单元（ALU）。其中ACC为累加器。MQ为寄存器，x为操作数寄存器
            各个寄存器所存放的各类操作数
            寄存器\运算     加法        减法        乘法        除法
            ACC            被加数和    被减数差     乘积高位    被除数和及余数  
            MQ                                     乘数及乘积低位 商 
            X              加数         减数        被乘数      除数

            加法操作过程
                【M】 → X
                【ACC】+【X】→ ACC
            减法操作过程
                【M】→ X
                【ACC】-【X】→ ACC
            乘法操作过程
                【M】→MQ
                【ACC】→X
                0→【ACC】
                【X】*【MQ】→ACC（高位）//MQ（低位）
            除法操作过程
                【M】→ X
                【ACC】/【X】→ MQ
                余数R在ACC中
    控制器
            控制器是计算机的神经中枢，由它指挥各部分自动协调的工作，具体而言他首先要命令存储器独处一条指令，称为取指过程（也成为取指阶段），接着它对这条指令进行分析，指出完成该指令学要进行工作，并按照寻址特征指明操作数的地址，称为分析过程，最后根据操作数地址以及指令的操作码完成某种操作，称为执行过程。
            控制器由程序计数器（pc），指令寄存器（IR）以及控制单元（CU）组成，pc用来存放当前欲执行指令的地址，他与主存的MAR之间有一条直接通路，且具有自动加一的功能，及可自动形成下一条指令的地址，IR用来存放当前指令，IR的内容来自主存的MDR。IR中的操作码（op（IR
            ））送至CU，记作OP（IR)→CU，用来分析指令，其地址码（ad（IR））作为操作数的地址送至存储器的MAR，记作Ad（IR）→MAR.CU用来分析当前指令所需的完成操作，并发出各种微操作命令序列，用以控制所有的对象。
    I/O
        i/o子系统包含各种i/o设备机器对应的接口。每一种i/o设备都由I/O接口与主机联系，它接收cu发出的各种控制命令。并完成相应的操作。
2，评价标准
    计算机硬件的主要技术指标
        1 机器字长
            机器字长是指cpu一次能处理数据的位数，通常与cpu的寄存器位数有关。
        2 存储容量 
            存储器的容量应该包括主存容量和辅存容量
            主存的容量是指主存中存放二进制代码的总数
                    存储容量= 存储单元个数*存储字长
        3 运算速度
            计算机的运算速度与许多因素有关，比如机器的主频、执行什么样的操作、主存本身的速度等有关。吉普森法综合考虑每条指令的执行时间以及他们在全部操作中所占的百分比
            现在机器的运算速度普遍使用单位时间内执行命令的平均条数来衡量，并用MIPS（百万条指令每秒）作为计量单位。也可以用CPI及执行一条指令所需要的时钟周期（机器主频的倒数）或用FLOPS（浮点运算次数每秒）来衡量运算速度。
3.层次结构 
        硬联逻辑级
        第零级是硬联逻辑级，这是计算机的内核，由门，触发器等逻辑电路组成。
        微程序级
        第一级是微程序级。这级的机器语言是微指令集，程序员用微指令编写的微程序，一般是直接由硬件执行的。
        传统机器级
        第二级是传统机器级，这级的机器语言是该机的指令集，程序员用机器指令编写的程序可以由微程序进行解释。
        操作系统级
        第三级是操作系统级，从操作系统的基本功能来看，一方面它要直接管理传统机器中的软硬件资源，另一方面它又是传统机器的延伸。
        汇编语言级
        第四级是汇编语言级，这级的机器语言是汇编语言，完成汇编语言翻译的程序叫做汇编程序。
        高级语言级
        第五级是高级语言级，这级的机器语言就是各种高级语言，通常用编译程序来完成高级语言翻译的工作。
        应用语言级
        第六级是应用语言级，    
        这一级是为了使计算机满足某种用途而专门设计的，因此这一级语言就是各种面向问题的应用语言。
        把计算机系统按功能分为多级层次结构，就是有利于正确理解计算机系统的工作过程，明确软件，硬件在计算机系统中的地位和作用。ssx
